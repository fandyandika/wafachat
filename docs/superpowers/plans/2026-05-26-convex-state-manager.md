# Convex State Manager Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Introduce Convex as the WaFaChat support-AI source of truth without changing the existing Order Trigger contract.

**Architecture:** Convex owns support-AI data and realtime panel state. n8n remains the connector layer and initially reaches Convex through the existing State Manager webhook adapter. The first implementation slice creates the Convex schema/functions and panel provider; n8n adapter wiring follows after Convex deployment URLs and auth secrets are available.

**Tech Stack:** Next.js 14 App Router, Convex, TypeScript, shadcn/ui, n8n HTTP Request nodes.
